## 정량 데이터
> 디렉터가 KPI 목표 대비 현재 상태를 판단하고, 핵심 재미가 구조적으로 전달되는지 확인하는 데 사용한다.

- 구간별 클리어율: (예: 스테이지1 95%, 스테이지2 72%, 스테이지3 41%) → 급격한 이탈이 발생하는 구간이 핵심 재미 도달 전인지 후인지를 판단한다.
- 주요 이탈 지점: (예: 튜토리얼 3단계, 보스1) → 이탈이 방향성 문제(재미 부재)인지 실행 문제(난이도/UX)인지 구분하는 단서다.
- 평균 세션 시간: → 의도한 세션 페이싱과 일치하는지 확인한다.
- 핵심 루프 완주율: → 핵심 재미에 도달한 플레이어 비율. Promise Delivery 판정의 핵심 근거다.
- 기타 측정 지표: (프로젝트별 자유 기재)

## 팀 내부 이슈
> 방향성과 무관하게 실행을 제약하는 요소. 디렉터가 Priority Matrix 조정 시 현실적 제약을 고려하는 데 사용한다.

- 현재 기술적 제약: (있다면) → Must Have 항목이 기술적으로 불가능한 경우 우선순위 재조정이 필요하다.
- 일정 리스크: (있다면) → Should Have를 Won't로 내리거나 스코프를 줄이는 판단의 근거다.
- 팀 내 의견 충돌: (있다면) → 디렉터가 방향성 기준으로 중재 판단을 내린다.
