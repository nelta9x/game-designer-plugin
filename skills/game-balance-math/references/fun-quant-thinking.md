# 재미를 수치로 사고하는 프레임 (절대 공식 금지)

이 문서는 "재미를 공식으로 계산"하려는 시도를 피하고,  
대신 **재미에 대해 수치적으로 어떻게 발상할지**를 정리한 실무 프레임이다.

핵심 원칙:
- 재미는 상수가 아니라 **가설(Hypothesis)** 이다.
- 수치는 정답이 아니라 **관찰 장치(Instrument)** 다.
- 단일 점수보다 **트레이드오프**를 먼저 본다.

## 1) 절대식 대신 "재미 가설"로 시작

아래 문장으로 시작하면 모호함이 줄어든다.

```text
우리가 올리고 싶은 재미는 [긴장감/성장쾌감/공정성/해방감]이고,
그 결과 플레이어가 [관찰 가능한 행동]을 보일 것이다.
실패 신호는 [명확한 반대 행동]이다.
```

예시:
- "챕터 경계 긴장감을 올리면, 보스 재도전은 늘되 챕터 이탈은 늘지 않아야 한다."
- "성장쾌감을 올리면, 세션 내 레벨업 빈도는 증가하되 후반 목표 소진은 유지되어야 한다."

## 2) 재미 축(Fun Axis)을 먼저 고른다

한 번에 1~2개 축만 선택한다. 축이 많아지면 판단이 무너진다.

| 재미 축 | 올리고 싶은 체감 | 대표 정량 신호 | 대표 정성 신호 |
|---|---|---|---|
| 긴장감 | 실패 직전 몰입, 재도전 의지 | p90 재시도, 보스 진입률, 세션 길이 | "빡세지만 해볼 만함" |
| 성장쾌감 | 성취의 연속감 | 레벨업 시간, 구간 통과 시간 | "쑥쑥 큰다" |
| 공정성 | 납득 가능한 결과 | TTK 분산, 편차, 실패 누적 분포 | "억까가 적다" |
| 해방감 | 벽 돌파 후 보상 체감 | 관문 통과율, 챕터 전환 잔존 | "뚫었을 때 시원함" |
| 신선도 | 패턴 피로 감소 | 재방문률, 다양한 빌드 사용률 | "매번 똑같지 않다" |

## 3) 지표는 "쌍"으로 설계한다

좋은 지표 1개 + 부작용 감시 지표 1개를 같이 둔다.

예시:
- 긴장감 상승 목표: `보스 재도전률` ↑
- 부작용 감시: `챕터 이탈률` ↑ 금지

권장 구성:
- 주지표(Primary) 2~3개
- 가드레일(Guardrail) 1~2개
- 실패 조건(Kill Criteria) 1~2개

## 4) 평균값 대신 분포를 본다

재미 불만은 평균보다 꼬리에서 먼저 터진다.

- 평균 + p90/p95를 기본으로 본다.
- 코호트(신규/중간/엔드), 구간(초중후반), 콘텐츠 티어별로 분리 본다.
- "전체 평균 개선"만으로 성공 판정하지 않는다.

## 5) AI 에이전트 사고 순서

사용자가 "재미"를 말하면 아래 순서로 답한다.

1. 재미 의도를 1문장으로 고정한다.
2. 상충 목표를 1개 이상 명시한다.  
   예: 긴장감 상승 vs 공정성 하락 위험
3. 조정 레버를 1~2개만 고른다.  
   예: ratio, 경계 위치, 실패 페널티
4. 주지표/가드레일/실패 조건을 제시한다.
5. 실패 시 롤백 또는 완충 레버를 함께 준다.

## 6) 답변 템플릿 (재미 요청 전용)

```text
재미 의도:
- [올리고 싶은 체감]

재미 가설:
- [변경]하면 [행동]이 나타나고, [부작용]은 제한된다.

관찰 신호:
- 주지표: [...]
- 가드레일: [...]

실패 조건:
- [조건 1]
- [조건 2]

조정/복구 레버:
- [레버 A], [레버 B]
```

## 7) 안티패턴

- "재미 = 클리어율 60%"처럼 단일 수치로 정의
- 평균값만 보고 롤아웃
- 목표 감정 없이 수치만 조정
- 실패 조건 없이 패치 강행
