# 성장 체감 의도 -> 곡선 선택 플레이북

이 문서는 사용자가 "어떤 느낌"을 말했을 때 곡선 모델을 빠르게 추천하기 위한 실무용 가이드다.

빠른 상담용 축약표가 필요하면 먼저 `intent-curve-one-pager.md`를 본다.

## 언제 이 문서를 여는가

- "레벨업이 지루하지 않게", "막히는 느낌 없게"처럼 체감 의도 중심 요청이 왔을 때
- 선형/지수/계단식 중 무엇을 먼저 제안할지 빠르게 정해야 할 때
- 추천 모델의 리스크와 검증 포인트를 같이 제시해야 할 때

## 빠른 추천 매트릭스

| 사용자 의도/느낌 | 1순위 모델 | 대안 모델 | 선택 이유 | 첫 조정 레버 |
|---|---|---|---|---|
| 예측 가능, 공정, 계산 쉬움 | 선형(Linear) | 낮은 파워(p≈1.1~1.4) | 다음 레벨 부담이 읽혀 불신이 적음 | base, 레벨당 증가량 |
| 초반 도파민, 후반도 목표 유지 | 지수(Exponential) | 중간 파워(p≈1.6~2.2) | 초반 빠른 성장 후 장기 동기 부여 | ratio(1.05~1.20), 구간별 ratio |
| 챕터별 관문/해방감, 이벤트 변속 | 계단식(Piecewise) | 선형+구간 배율 | 구간 경계에서 의도적 체감 전환 가능 | 구간 경계, 구간별 기울기 |
| 온보딩은 빠르게, 중반 안정화 | 로지스틱(Sigmoid) | 낮은 파워 + 캡 | 초반 허들 완화, 중반 과속 방지 | k, x0, 상/하한 |
| 후반 과성장 억제, 시스템 보호 | 체감(Diminishing) | 로지스틱 캡 | 투자 체감으로 메타 폭주 억제 | k(반감점), 상한값 |

## 모델별 적용 사례

### 1) 선형(Linear)

추천 상황:
- 유저가 "억울하지 않게", "실력으로 올라가는 느낌"을 원할 때
- PvP/MMR, 시즌 패스, 튜토리얼 구간처럼 예측 가능성이 중요한 시스템

실무 사례:
- 시즌 초 1~20레벨에서 레벨당 요구 XP를 거의 일정하게 유지해 이탈을 줄임
- 일일 성장 미션에 선형 누적치 사용해 달성 감각을 명확히 제공

주의:
- 장기 콘텐츠에서는 후반 목표 약화 가능
- 해결: 1~20 선형 + 21+ 완만 파워/계단식 하이브리드

### 2) 지수(Exponential)

추천 상황:
- 유저가 "초반은 빨리, 후반은 오래 붙잡는" 느낌을 원할 때
- 장기 루프가 중요한 MMORPG/아이들/엔드게임 중심 구조

실무 사례:
- 신규 유저 온보딩 구간은 ratio를 낮게 유지하고, 엔드게임에서만 ratio를 점진 상향
- 시즌 후반 랭크 경쟁에서 레벨 업 비용을 지수로 올려 상위권 분리

주의:
- ratio 과대 설정 시 중반 급벽으로 이탈 증가
- 해결: ratio를 구간 분리(예: 1.08 -> 1.12)하거나 중반은 파워로 완화

### 3) 계단식(Piecewise)

추천 상황:
- 유저가 "챕터 넘어갈 때 확실한 긴장감"을 원할 때
- 캠페인/시즌/레이드 티어처럼 구간 경계가 콘텐츠 의미와 맞물릴 때

실무 사례:
- 챕터 시작 직전 레벨업 부담을 살짝 올리고, 챕터 클리어 후 보상 구간에서 다시 완화
- 이벤트 기간에 특정 구간만 성장률 버프를 적용해 회귀 유저 온보딩 강화

주의:
- 경계 체감이 과하면 "갑자기 막힌다"는 피드백 발생
- 해결: 경계 전후 2~3레벨 완충 구간을 넣고 급격한 기울기 차이를 축소

## 안티패턴 -> 복구 레버

| 문제 신호 | 흔한 원인 | 즉시 복구 레버 |
|---|---|---|
| "중반부터 갑자기 막힘" | 지수 ratio 과대, 계단 경계 급변 | ratio 하향, 경계 완충(2~3레벨), 보상 상향 |
| "너무 단조롭고 성취감 약함" | 선형만 장기 구간까지 유지 | 중후반에 파워/계단식으로 전환 |
| "구간마다 난이도 들쭉날쭉" | Piecewise 경계 설계 미흡 | 구간 간 기울기 차이 제한, 전이 구간 추가 |
| "상위권만 더 빨라짐" | 보상과 성장률 동시 상향 | 성장률 상향 폭 축소, 체감 캡 적용 |

## 추천 시 답변 포맷(에이전트용)

사용자가 체감 의도를 말하면 아래 형식으로 답한다.

```text
의도 요약:
- [사용자가 원하는 느낌]

추천 모델(우선순위):
1) [모델 A]
2) [모델 B]

왜 이 모델인가:
- [체감/시스템 이유]

리스크:
- [과성장/정체/경계 충격 등]

첫 조정 레버:
- [ratio, p, 구간 경계, k]

검증 지표:
- [레벨업 시간, TTK, TTE, 첫 시도 클리어율]
```
